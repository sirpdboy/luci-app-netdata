XSS Protection

This dashboard is about to render data from server:

To protect your privacy, the dashboard will check all data transferred for cross site scripting (XSS).
This is CPU intensive, so your browser might be a bit slower.

If you trust the remote server, you can disable XSS protection.
In this case, any remote dashboard decoration code (javascript) will also run.

If you don't trust the remote server, you should keep the protection on.
The dashboard will run slower and remote dashboard decoration code will not run, but better be safe than sorry...

打印此netdata仪表板

netdata dashboards cannot be captured, since we are lazy loading and hiding all but the visible charts.
To capture the whole page with all the charts rendered, a new browser window will pop-up that will render all the charts at once. The new browser window will maintain the current pan and zoom settings of the charts. So, align the charts before proceeding.

This process will put some CPU and memory pressure on your browser.
For the netdata server, we will sequencially download all the charts, to avoid congesting network and server resources.
Please, do not print netdata dashboards on paper!

仪表板说明

  • 滑鼠介面
  • 触控介面

滑鼠移至 / 暂留

将鼠标移到图表上，在图表的图例处显示鼠标下的时间戳值（图表还将突出显示图表上的点）.
所有其他可见图表也将显示并突出显示相同时间戳的值.

拖曳图表内容

通过按下鼠标左键并移动鼠标来拖动图表的内容，以水平平移图表.
所有的图表都会在你把图表放在一边后很快出现（这个小小的延迟是出于设计：它加速了你的浏览器，让你专注于你正在探索的东西）.
平移图表后，所有图表的自动刷新将停止。为了再次启用它， double click a panned chart.

连点两下

双击图表可将所有图表重置为默认的自动刷新状态。

SHIFT + 拖曳

在按下 SHIFT 键，在图表内容上按鼠标左键，然后移动鼠标以选择区域，以放大。其他图表也将紧随其后。缩放分两个阶段执行:
  • 已加载的图表内容被缩放（低分辨率）
  • 从netdata服务器传输新数据，以刷新可能包含更多细节的图表.
图表缩放后，所有图表的自动刷新将停止。再次启用它, double click a zoomed chart.

反白时间轴

在按下 ALT 键，在图表内容上按鼠标左键，然后移动鼠标以选择区域。所有图表上都将突出显示所选内容。

SHIFT + 滑鼠滚轮

在按下 SHIFT 键，鼠标指针位于图表内容上方，滚动鼠标滚轮以放大或缩小。这种缩放与鼠标指针下方的中心对齐。其他图表也将紧随其后。
图表缩放后，所有图表的自动刷新将停止。再次启用它, double click a zoomed chart.

图例操作

单击标注的标签或值，将选择/取消选择该标注。
可以按任意SHIFT键或CONTROL键，然后单击图例标签或值，以选择/取消选择多个标注。

Single Tap

点击图表内容，在其图例处显示所点击时间戳的值（图表还将突出显示图表上的点）.
所有其他可见图表也将显示并突出显示相同时间戳的值.

Drag Chart Contents

触摸并拖动图表内容以水平平移
所有的图表都会在你把图表放在一边后很快出现（这个小小的延迟是出于设计：它加速了你的浏览器，让你专注于你正在探索的东西）.
平移图表后，所有图表的自动刷新将停止。为了再次启用它， double tap a panned chart.

Double Tap

双击图表可将所有图表重置为默认的自动刷新状态

Zoom (does not work on firefox and IE/Edge)

用两根手指放大或缩小。
图表缩放后，所有图表的自动刷新将停止。再次启用它, double click a zoomed chart.

图例操作

轻触标注的标签或值，将选择/取消选择该标注。

netdata 仪表板选项

这里是浏览器的设定，每个浏览者都可以拥有自己的设定。这里的设定不会影响 netdata 本身的运作。
设定会立即生效，并且储存在您浏览器的本地储存中 (除了重新整理的 在焦点上/总是 这两个选项以外)。
若要将所有选项重置回预设值 (包括图表大小)，请点选 这里。
  • 效能
  • 同步
  • 视觉
  • 地区
何时要重新整理图表？
当设置为 在焦点上, 如果页面/选项卡没有用户的焦点，图表将停止更新。当设置为 总是, 图表将始终刷新。设定为 在焦点上 当您不关注此页面时，可以降低浏览器的CPU要求（并扩展笔记本电脑和平板电脑的电池）。设置为总是 在另一个窗口上工作（即更改某些设置）并在此窗口中自动刷新图表。
要显示那些图表维度？
当设置为 非零, 将所有值（在当前视图中）设置为零的维度将不会从netdata服务器传输（除非图表的所有维度都为零，在这种情况下，此设置不起任何作用-所有维度都将传输并显示）。当设置为 所有，所有尺寸将始终显示。将其设置为非零 要降低netdata和浏览器之间传输的数据，请降低浏览器的CPU要求（要绘制的线更少），并增加图例的焦点（图例处的条目更少）
如何处理隐藏图表？
当设置为 消去, 不在浏览器当前视口中的图表（位于页面可见区域的上方或下方）将在再次可见时销毁并重新创建。当设置为 隐藏，不可见的图表将被隐藏，以简化DOM并加快浏览器速度。将其设置为消去 ，以降低浏览器的内存要求。将其设置为隐藏 用于在页面滚动时更快地恢复图表。
如何处理页面卷动？
当设置为 同步, 滚动后，将立即检查图表的可见性。在速度较慢的计算机上，这可能会影响页面滚动的平滑度。要在滚动结束时更新页面，请将其设置为异步。将其设置为“同步”，以便在滚动时立即更新图表。将其设置为非同步 在速度较慢的计算机上进行更平滑的页面滚动
采用何种图表更新模式？
当设置为 并行, 可见图表并行刷新（所有查询并行发送到netdata server）并异步呈现。当设置为 顺序图表会一个接一个地刷新。如果您的浏览器可以处理它，请将其设置为并行（大多数现代浏览器都可以），如果您在较旧/较慢的计算机上工作，请将其设置为顺序。
重新整理时是否需要同步所有图表？
当设置为 重新同步, 仪表板将尝试重新同步所有图表，以便它们同时刷新。当设置为 尽力而为 ，每个图表可能会刷新，但与其他图表有一点时差。通常情况下，仪表板会同时开始刷新它们，但根据计算机的速度和网络延迟，图表开始有轻微的时差。将此设置为重新同步 将在每次更新时尝试重新同步图表。将其设置为尽力而为 可能会降低浏览器和网络的压力
是否悬停在图表上时同步更新所有图表？
启用时，一个图表上的选择将自动同时在所有其他可见图表上选择，并且所有可见图表的图例将更新以显示所选值。禁用时，将仅选择引起用户注意的图表。使it部门能够更好地了解数据。禁用它，如果你是在一个非常缓慢的计算机上，实际上无法做到这一点.
是否同步移动与缩放所有图表？
启用后，图表上的平移和缩放操作将复制到所有图表（即使是当前不可见的图表，当然只有当它们变为可见时）。禁用时，平移和缩放操作将不会传播到其他图表.
使用那一种布景主题？
Netdata有两个主题： 深色 (the default) and 浅色.
切换此选项将重新加载仪表板.
您需要说明吗？
Netdata可以在某些方面显示一些帮助，以帮助您使用仪表板。如果所有这些气球都困扰您，请使用此开关禁用它们。
切换此选项将重新加载仪表板.
是否启用在移动与缩放时同时填补数据？
当设置为 Pad 图表将在可见区域之前和之后填充更多数据，从而给人以整个数据库已加载的印象。此填充仅在对图表进行第一次平移或缩放操作后发生（最初，所有图表都只有可见数据. 当设置为 不需填补 即使在第一次平移和缩放操作之后，也只会从netdata服务器传输可见数据。
是否在图表上启用贝兹曲线？
当设置为 平滑 支持它的图表库将绘制平滑曲线，而不是简单的直线来连接点
Keep in mind dygraphs, netdata仪表板中的主图表库只能平滑折线图。它无法平滑面积或堆叠图表。 当设置为 Rough, 此设置可以降低浏览器消耗的CPU资源。
随着图表的更新，这些设置将逐渐应用。要强制执行，请立即刷新仪表板.
是否启用自动缩放单位选择？
当设置为 缩放单位 the values shown will dynamically be scaled (e.g. 1000 kilobits will be shown as 1 megabit). Netdata can auto-scale these original units: kilobits/s, kilobytes/s, KB/s, KB, MB, and GB. 当设置为 固定单位 all the values will be rendered using the original units maintained by the netdata server.
使用何种温度单位？
Set the temperature units of the dashboard.
是否将秒转换为时间？
当设置为 时间, 当前的图表 秒 will show DDd:HH:MM:SS. 当设置为 秒, 将显示原始秒数.
是否显示浏览者本地时间或是伺服器时间？
当设置为 本地时间, 图表将使用你的浏览器本地时间。 当设置为 Server Time the charts will use the server time.
Set it to 本地时间 要有一个通用的时间参考，无论服务器在哪里，使用哪个时区（所有仪表板都将报告您的本地时间）. Set it to 伺服器时间 当您需要将netdata图表与服务器日志文件进行比较时.
